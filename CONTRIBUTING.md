# Contributing

Thanks for stopping by. This is a small zero-dependency project — keep patches
small and focused. No formal issue template, just be explicit about what you're
fixing and attach a minimal reproducer if it's a bug.

## Getting started

```bash
git clone https://github.com/andromanpro/ai-conversation-viz.git
cd ai-conversation-viz
npm test          # runs tests/run.js against src/ (ES modules, zero deps)
npm run build     # rebuilds dist/ai-conversation-viz.js (IIFE bundle)
```

Then open `index.html` in a browser, or `standalone.html` if you want to test
the bundled version without a server.

## Project layout

```
src/
├── core/       Pure logic — parser, adapters, graph, physics, i18n.
│               No DOM, no window.* — must work in Node for tests.
├── view/       Rendering — Canvas 2D, WebGL, starfield, particles, topics.
│               DOM allowed, but no event handlers.
├── ui/         Event handlers, DOM manipulation, toggles, panels.
│               Everything that reacts to user clicks lives here.
├── 3d/         Three.js-based 3D renderer (standalone entry).
├── main.js     2D entry point — wires core + view + ui together.
└── embed.js    npm-package API — mount(el, { jsonl }).

tests/run.js   Hand-rolled test runner (no Jest). All imports from src/.
dist/          Generated by build.cjs. Do not edit by hand.
samples/       Example JSONL files.
```

## Code style

- **2 spaces, LF, single quotes**, enforced via `.editorconfig` + `.gitattributes`.
- **ES2020+ modules everywhere** in `src/`. No CommonJS, no TypeScript.
- **JSDoc** on public exports (`@param`, `@returns`). Internal helpers may skip it.
- **No runtime `console.log`**. `console.warn` is OK for real errors.
- **No TODOs** without a linked issue. If you need a marker, prefer a proper
  issue.

## Adding strings

All user-visible strings go through `src/core/i18n.js`:

```js
import { t } from '../core/i18n.js';
_btn.title = t('tip.freeze_off');
```

Add both `en` and `ru` entries in the DICT at the top of `i18n.js`. For HTML
static text use `data-i18n="key"`, `data-i18n-title="key"`,
`data-i18n-placeholder="key"`, or `data-i18n-aria="key"`.

## Architecture boundaries

- `src/core/*` must not reach into `window`, DOM, or `ui/view/*`. Pure
  functions. If you need UI state inside core, pass it as a parameter.
- `src/view/*` may touch the DOM (canvas, style), but must not install event
  handlers — those belong in `ui/`.
- Circular imports are forbidden. Run `npm test` — the module-namespace bundler
  topologically sorts imports, cycles fail the build.

## index.html and standalone.html must stay in sync

There are two HTML entry points:

- **`index.html`** — for development, loads `src/main.js` as ES modules.
- **`standalone.html`** — for `file://` use, loads `dist/ai-conversation-viz.js` (the IIFE bundle).

Both render the same UI. Any DOM change (new button, new panel, new id, new
`data-i18n*` attribute) must be applied to **both** files in the same PR.

Quick check before committing:

```bash
diff <(grep -E "btn-|data-i18n" index.html) <(grep -E "btn-|data-i18n" standalone.html)
```

Only legitimate diff: `header.subtitle_force` vs `header.subtitle_standalone`
in the subtitle div. Anything else is a sync bug.

## Pull requests

- One concern per PR. If you find three things, send three PRs.
- Run `npm test && npm run build` locally. All 122+ tests must pass.
- Rebuild `dist/ai-conversation-viz.js` before committing (CI does not yet do
  this; this will change once a GitHub Actions workflow lands).
- Commit messages: `vX.Y.Z: short summary` for releases, imperative mood for
  smaller commits (`fix X`, `add Y`).
- **Do not** add `Co-Authored-By: Claude` to commits.
